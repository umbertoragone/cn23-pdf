When filling out a **CN23**, one of the most confusing parts is the customs code field. Some people look for a **TARIC code**, others for an **HS number**, and others just call it the product customs code.

In the **[CN23.it](/)** generator, the field is labeled **HS tariff #**, which is the HS tariff number associated with each parcel line item. Understanding which code belongs there helps you avoid vague declarations, reduce manual checks, and get to the final PDF faster.

## TARIC and HS are not the same thing

In simple terms:

- **HS** is the base customs classification used internationally
- **TARIC** is a more detailed layer used in the European customs framework

That is why many shippers search for a `TARIC code` when the first thing they really need is the **correct HS number**. If your shipping workflow or broker requires a more specific code, you should still confirm the exact format against the official documentation you rely on.

## How to find the right HS number

The fastest method is to start from the actual product, not the marketing name.

1. Describe the item precisely.
   Example: not `accessory`, but `leather phone case`.
2. Check the material and the main function.
   Similar products can fall under different codes if the material or use changes.
3. Review the invoice, product sheet, or supplier documentation.
   The technical description you need is often already there.
4. Search the customs nomenclature you use for shipments.
   In the EU, many people start from the TARIC database to identify the correct heading.
   A practical reference is [tariffnumber.com](https://www.tariffnumber.com), which indexes the European database and lets you search for a harmonized HS code starting from product descriptions in languages such as English, French, and German.
5. Compare nearby classifications and choose the most specific match.
   If a category still feels too broad, you probably have not narrowed it down enough.

## A quick check before filling the CN23

Before entering the code in the form, make three fast checks:

- the item description and the code refer to the same product
- the declared value makes sense for that item
- the line weight is consistent with what is actually inside the parcel

If one of those three points looks wrong, the issue is usually not the PDF. It is usually the original classification or product description.

## How to generate the PDF in 2 minutes

Once you have the right code, the workflow in the generator is straightforward:

1. Enter sender and recipient details.
2. Add each item with a clear description, quantity, weight, value, and **HS tariff #**.
3. Review the totals.
4. Check the preview.
5. Download the PDF ready for printing.

If you ship similar goods often, you can also reopen or share the same URL with most fields already filled in and only update the shipment-specific details.

## Summary

Filling a CN23 properly is not just about writing `gift` or `sample` and moving on. The key step is matching each line item with a clear description and a **consistent HS number**. Once that is done, generating the PDF really can take just a couple of minutes.

The final manual review still matters: customs codes, destination-country requirements, and carrier instructions should always be verified by the sender before shipping.
